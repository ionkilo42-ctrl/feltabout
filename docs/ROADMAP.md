# Feltabout Roadmap

This roadmap is intentionally conservative. It describes the direction the current repository supports, not a promise of delivery dates.

## Current Focus

### v0.1.x alpha

- tighten the individual reflection and conversation-prep loop
- reduce ambiguity between active MVP 1 code and legacy/reference code
- strengthen safety checks, eval coverage, and refusal behavior
- improve self-hosting documentation and contributor onboarding
- stabilize the primary web and API workflow

## Next

### Short-term product and platform work

- make the core reflection flow easier to run locally with fewer hidden assumptions
- improve docs around environment variables, auth modes, and persistence choices
- expand regression coverage for safety gating, prompt outputs, and route behavior
- refine the Aimee/guide surfaces without turning them into companion or therapy framing
- add clearer screenshots and demo assets for public OSS review

## Later

### Potential post-alpha exploration

- stronger auditability around generated outputs and safety events
- better export/history tooling for user-owned reflection data
- cleaner packaging for self-hosted deployments
- more deliberate handling of experimental voice infrastructure

## Explicitly Out of Scope for MVP 1

- therapy or mental health treatment features
- diagnosis or psychiatric assessment
- crisis intervention beyond detection and routing
- social feeds, gamification, or emotional dependency loops
- live multi-party mediation as a default product path
- payment systems or growth mechanics

## Decision Rule

If a proposed roadmap item makes Feltabout look more like a therapist, a crisis hotline, a social app, or a coercion tool, it does not belong on the roadmap.
