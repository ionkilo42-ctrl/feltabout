# OSS Application Notes

These notes are intended for a Codex for Open Source application and stay grounded in the current repository.

## Why This Repository Qualifies

Feltabout is an open-source, self-hostable codebase for safety-conscious reflection and difficult-conversation preparation. It combines a FastAPI backend, a Next.js web app, an Expo mobile prototype, tests, CI, and explicit product-safety documentation. The project explores how AI can help people prepare language more carefully without claiming therapy, diagnosis, or crisis care.

## How We Will Use API Credits

We will use API credits to improve the active reflection pipeline in `services/api`, evaluate prompt outputs against safety cases, test provider abstraction quality, refine refusal behavior for coercive or crisis-adjacent inputs, and generate better open-source documentation examples. Credits would support development, evals, and safer iteration rather than growth claims or production-scale usage.

## Anything Else We Should Know

This repository is intentionally safety-first and non-clinical. The codebase already separates safety, reflection, and facilitation concerns, and the docs make clear that Feltabout is not therapy, diagnosis, or crisis support. The current maintainer priority is tightening the MVP 1 vertical slice, improving tests and docs, and making the project easier for contributors to inspect, run, and critique honestly.
